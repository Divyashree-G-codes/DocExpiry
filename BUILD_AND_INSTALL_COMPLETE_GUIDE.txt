TROUBLESHOOTING
================================================================================

ERROR: "JAVA_HOME is set to an invalid directory"
FIX:   In CMD.EXE, run these commands before building:
       set JAVA_HOME=
       gradlew.bat clean assembleDebug

ERROR: "gradlew.bat" not recognized
FIX:   Make sure you're in the project directory:
       cd C:\Users\Shara\AndroidStudioProjects\DocExpiry
       Then try: .\gradlew.bat clean assembleDebug

ERROR: "adb is not recognized" (during install step)
FIX:   You can skip adb and use Method 1 (Drag & Drop) instead
       Or set Android SDK path in environment variables:
       setx PATH "%PATH%;C:\Users\Shara\AppData\Local\Android\Sdk\platform-tools"

ERROR: Build fails with "Compilation error"
FIX:   Check build_full.log for details:
       type build_full.log | findstr /I "error"

SUPPORT / ISSUES
================================================================================

If build fails:
  1. Check clean.log or build_full.log for error details
  2. Try clearing gradle cache: gradlew.bat --stop
  3. Delete build folder and rebuild: gradlew.bat clean assembleDebug

If app crashes on device:
  1. View device logs: adb logcat | findstr docexpiry
  2. Check error messages in logcat output
  3. Ensure device Android version matches (SDK 30+)

If you need help with specific changes:
  - All code files are in: app/src/main/java/com/example/docexpiry/
  - All layout files are in: app/src/main/res/layout/
  - All drawables are in: app/src/main/res/drawable/
